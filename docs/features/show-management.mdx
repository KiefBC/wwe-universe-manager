## Overview

The Show Management feature allows you to create, organize, and manage wrestling shows within your WWE Universe. Each show represents a wrestling program with its own identity, roster, and storylines.

## Creating a Show

<Steps>
  <Step title="Navigate to Create Show">
    Click the "CREATE SHOW" button from the main dashboard
  </Step>
  <Step title="Fill in Show Details">
    Provide a name and description for your show:
    - **Name**: The show's title (e.g., "Monday Night Raw", "SmackDown Live")
    - **Description**: A brief description of the show's theme or purpose
  </Step>
  <Step title="Save the Show">
    Click "CREATE SHOW" to save your new show to the database
  </Step>
</Steps>

## Database Schema

Shows are stored with the following structure:

```sql
CREATE TABLE shows (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,
  description TEXT NOT NULL,
  created_at DATETIME,
  updated_at DATETIME
);
```

## API Reference

### Create Show

<CodeGroup>

```rust Tauri Command
#[tauri::command]
pub fn create_show(
    state: State<'_, DbState>, 
    show_data: ShowData
) -> Result<Show, String>
```

```rust Request Type
pub struct ShowData {
    pub name: String,
    pub description: String,
}
```

```rust Response Type
pub struct Show {
    pub id: i32,
    pub name: String,
    pub description: String,
}
```

</CodeGroup>

### Get Shows

Retrieve all shows from the database:

```rust
#[tauri::command]
pub fn get_shows(state: State<'_, DbState>) -> Result<Vec<Show>, String>
```

## Best Practices

<AccordionGroup>
  <Accordion title="Naming Conventions">
    - Use clear, descriptive show names
    - Avoid special characters that might cause issues
    - Consider branding consistency across your universe
  </Accordion>
  
  <Accordion title="Descriptions">
    - Keep descriptions concise but informative
    - Include the show's theme or target audience
    - Update descriptions as your show evolves
  </Accordion>
  
  <Accordion title="Organization">
    - Plan your show hierarchy before creating
    - Consider how shows will interact in your universe
    - Think about scheduling and conflicts
  </Accordion>

## Common Issues

<Warning>
  **Empty Fields**: Make sure both name and description are filled before submitting the form.
</Warning>

<Info>
  **Duplicate Names**: The system allows duplicate show names, but it's recommended to use unique names for clarity.
</Info>
