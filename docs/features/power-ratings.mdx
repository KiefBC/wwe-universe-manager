## Overview

The Power Ratings System is a sophisticated character development framework that quantifies wrestler abilities across six key attributes. This system transforms basic wrestler profiles into detailed character assessments that can be used for booking decisions, storyline development, and realistic match outcomes.

## Character Archetypes

The power ratings system naturally creates distinct wrestler archetypes:

### The Powerhouse
- **High**: Strength (90+), Stamina (75+)
- **Low**: Speed (60-), Agility (50-)
- **Examples**: Braun Strowman, Brock Lesnar
- **Booking**: Dominant squash matches, David vs Goliath storylines

### The High-Flyer
- **High**: Agility (85+), Speed (80+)
- **Low**: Strength (65-), Stamina (70-)
- **Examples**: Rey Mysterio, Ricochet
- **Booking**: Aerial showcases, underdog storylines

### The Technical Master
- **High**: Technique (90+), Stamina (80+)
- **Low**: Charisma (70-), Strength (75-)
- **Examples**: Daniel Bryan, Kurt Angle
- **Booking**: Wrestling clinics, submission specialists

### The Complete Package
- **High**: All attributes 75+
- **Low**: None below 70
- **Examples**: John Cena, Roman Reigns
- **Booking**: Main event players, championship contenders

### The Specialist
- **High**: One attribute 95+
- **Low**: Several attributes 60-
- **Examples**: Ultimate Warrior (Charisma 95, Technique 45)
- **Booking**: Character-driven storylines

## Rating Management Interface

### Individual Wrestler Editing

Power ratings are managed through the Wrestler Details Window:

```rust
#[component]
pub fn PowerRatingsSection(wrestler: ReadSignal<Option<Wrestler>>) -> impl IntoView {
    let (editing_ratings, set_editing_ratings) = create_signal(false);
    let (temp_ratings, set_temp_ratings) = create_signal(PowerRatings::default());
    
    let save_ratings = move |_| {
        if let Some(w) = wrestler.get() {
            let ratings = temp_ratings.get();
            spawn_local(async move {
                match update_wrestler_power_ratings(w.id, ratings).await {
                    Ok(updated_wrestler) => {
                        set_editing_ratings.set(false);
                        // Update wrestler signal
                    }
                    Err(e) => log::error!("Failed to update ratings: {}", e),
                }
            });
        }
    };
    
    view! {
        <div class="bg-slate-700 rounded-lg p-4">
            <div class="flex justify-between items-center mb-4">
                <h3 class="text-lg font-semibold text-yellow-400">"Power Ratings"</h3>
                <button 
                    on:click=move |_| set_editing_ratings.set(!editing_ratings.get())
                    class="btn btn-sm btn-secondary"
                >
                    {move || if editing_ratings.get() { "Cancel" } else { "Edit" }}
                </button>
            </div>
            
            <div class="grid grid-cols-2 gap-4">
                <RatingDisplay 
                    label="Strength" 
                    value=move || wrestler.get().and_then(|w| w.strength).unwrap_or(50)
                    editing=editing_ratings
                    on_change=move |val| {
                        let mut ratings = temp_ratings.get();
                        ratings.strength = val;
                        set_temp_ratings.set(ratings);
                    }
                />
                
                <RatingDisplay 
                    label="Speed" 
                    value=move || wrestler.get().and_then(|w| w.speed).unwrap_or(50)
                    editing=editing_ratings
                    on_change=move |val| {
                        let mut ratings = temp_ratings.get();
                        ratings.speed = val;
                        set_temp_ratings.set(ratings);
                    }
                />
                
                // ... similar for other attributes
            </div>
            
            <Show when=editing_ratings>
                <div class="mt-4 flex gap-2">
                    <button on:click=save_ratings class="btn btn-primary btn-sm">
                        "Save Changes"
                    </button>
                </div>
            </Show>
        </div>
    }
}
```

### Visual Rating Display

Each attribute is displayed with visual indicators:

```rust
#[component]
pub fn RatingDisplay(
    label: &'static str,
    value: impl Fn() -> i32 + 'static,
    editing: ReadSignal<bool>,
    on_change: impl Fn(i32) + 'static,
) -> impl IntoView {
    let rating_color = move || {
        match value() {
            90..=100 => "text-purple-400", // Legendary
            80..=89 => "text-red-400",     // Excellent  
            70..=79 => "text-orange-400",  // Very Good
            60..=69 => "text-yellow-400",  // Good
            50..=59 => "text-green-400",   // Average
            _ => "text-gray-400",          // Developing
        }
    };
    
    let rating_tier = move || {
        match value() {
            90..=100 => "Legendary",
            80..=89 => "Excellent",
            70..=79 => "Very Good", 
            60..=69 => "Good",
            50..=59 => "Average",
            _ => "Developing",
        }
    };
    
    view! {
        <div class="flex justify-between items-center">
            <div>
                <span class="font-medium">{label}</span>
                <div class="text-xs text-slate-400">{rating_tier}</div>
            </div>
            
            <div class="flex items-center gap-2">
                <Show 
                    when=editing
                    fallback=move || view! {
                        <span class=move || format!("font-bold text-lg {}", rating_color())>
                            {value}
                        </span>
                    }
                >
                    <input 
                        type="range"
                        min="1"
                        max="100"
                        prop:value=value
                        on:input=move |ev| on_change(event_target_value(&ev).parse().unwrap_or(50))
                        class="range range-primary range-sm"
                    />
                    <span class=move || format!("font-bold text-sm w-8 {}", rating_color())>
                        {value}
                    </span>
                </Show>
            </div>
        </div>
    }
}

// Example booking logic
pub fn calculate_match_probability(
    wrestler1: &Wrestler, 
    wrestler2: &Wrestler,
    match_type: &str
) -> f32 {
    let mut w1_advantage = 0.0;
    let mut w2_advantage = 0.0;
    
    match match_type {
        "Singles" => {
            // Balanced consideration of all attributes
            w1_advantage += wrestler1.technique.unwrap_or(50) as f32 * 0.3;
            w1_advantage += wrestler1.strength.unwrap_or(50) as f32 * 0.2;
            w1_advantage += wrestler1.stamina.unwrap_or(50) as f32 * 0.2;
            w1_advantage += wrestler1.speed.unwrap_or(50) as f32 * 0.15;
            w1_advantage += wrestler1.agility.unwrap_or(50) as f32 * 0.15;
            
            // Similar calculation for wrestler2
            // ...
        }
        "Ladder Match" => {
            // Agility and strength more important
            w1_advantage += wrestler1.agility.unwrap_or(50) as f32 * 0.4;
            w1_advantage += wrestler1.strength.unwrap_or(50) as f32 * 0.3;
            // ...
        }
        // Other match types...
    }
    
    // Return probability as percentage
    w1_advantage / (w1_advantage + w2_advantage)
}
```

                ("Technique", w1.technique.unwrap_or(50), w2.technique.unwrap_or(50)),
            ]
        } else {
            vec![]
        }
    };
    
    view! {

                    key=|(attr, _, _)| attr.to_string()

### Overall Rating Calculation

A composite rating helps with overall wrestler assessment:

```rust
pub fn calculate_overall_rating(wrestler: &Wrestler) -> i32 {
    let strength = wrestler.strength.unwrap_or(50) as f32;
    let speed = wrestler.speed.unwrap_or(50) as f32;
    let agility = wrestler.agility.unwrap_or(50) as f32;
    let stamina = wrestler.stamina.unwrap_or(50) as f32;
    let charisma = wrestler.charisma.unwrap_or(50) as f32;
    let technique = wrestler.technique.unwrap_or(50) as f32;
    
    // Weighted average based on wrestling importance
    let overall = (
        technique * 0.25 +      // Most important for match quality
        stamina * 0.20 +        // Essential for consistency
        charisma * 0.20 +       // Critical for main event status
        strength * 0.15 +       // Physical presence
        speed * 0.10 +          // Exciting offense
        agility * 0.10          // Innovation and excitement
    );
    
    overall.round() as i32
}
```

pub fn is_title_ready(wrestler: &Wrestler, title: &Title) -> bool {

### Match Quality Prediction

Use ratings to predict match quality for booking decisions:

```rust
pub fn predict_match_quality(participants: &[&Wrestler]) -> i32 {
    if participants.is_empty() { return 0; }
    
    let avg_technique: f32 = participants.iter()
        .map(|w| w.technique.unwrap_or(50) as f32)
        .sum::<f32>() / participants.len() as f32;
        
    let avg_charisma: f32 = participants.iter()
        .map(|w| w.charisma.unwrap_or(50) as f32)
        .sum::<f32>() / participants.len() as f32;
    
    // Match quality based on in-ring ability and star power
    let quality = (avg_technique * 0.7) + (avg_charisma * 0.3);
    
    quality.round() as i32
}
```
