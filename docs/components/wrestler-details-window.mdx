## Overview

The WrestlerDetailsWindow component displays detailed wrestler information in a separate Tauri window. This provides a dedicated space for viewing wrestler profiles without cluttering the main dashboard interface.

## Window Architecture

### Separate Window Design

The wrestler details are displayed in a completely separate application window, not a modal or overlay:

## Technical Implementation

### Window Creation

The window is created via a Tauri command from the main dashboard:

```rust
// Tauri backend command
#[tauri::command]
pub async fn open_wrestler_window(app: AppHandle) -> Result<(), String> {
    let window = WebviewWindowBuilder::new(
        &app,
        "wrestler-details",
        WebviewUrl::App("index.html#wrestler".into())
    )
    .title("Wrestler Details")
    .inner_size(800.0, 600.0)
    .resizable(true)
    .build()
    .map_err(|e| e.to_string())?;

    Ok(())
}
```

### Frontend Detection

The component detects it's in a separate window by checking the URL hash:

```rust
#[component]
pub fn App() -> impl IntoView {
    let hash = window().location().hash().unwrap_or_default();
    
    if hash == "#wrestler" {
        view! { <WrestlerDetailsWindow /> }
    } else {
        view! { <Dashboard /> }
    }
}
```

## Window Management

### Opening the Window

From the main dashboard:

<Steps>
  <Step title="Click Wrestlers Button">
    User clicks the "Wrestlers" button on the dashboard
  </Step>
  <Step title="Tauri Command Execution">
    Frontend calls the `open_wrestler_window` Tauri command
  </Step>
  <Step title="Window Creation">
    Tauri creates a new window with the wrestler URL hash
  </Step>
  <Step title="Component Rendering">
    App component detects the hash and renders WrestlerDetailsWindow
  </Step>
</Steps>

### Window Properties

The wrestler window is configured with:

- **Size**: 800x600 pixels (resizable)
- **Title**: "Wrestler Details"
- **URL**: `index.html#wrestler`
- **Independence**: Separate from main window
