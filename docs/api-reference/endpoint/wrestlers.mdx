## Overview

The Wrestlers API provides comprehensive wrestler management functionality, including basic CRUD operations, enhanced wrestler profiles with power ratings, and detailed biographical information.

## Core Operations

### Get All Wrestlers

<CodeGroup>

```rust Frontend (Leptos)

```


## Error Handling

All wrestler operations return `Result<T, String>` where the error string contains details about what went wrong:

- **Database connection errors**: "Failed to get database connection"
- **Wrestler not found**: "Wrestler with id {id} not found"
- **Validation errors**: "Invalid wrestler data provided"
- **Foreign key constraints**: "Cannot delete wrestler - referenced by other records"

## Usage Notes

- Power ratings should be between 1-100
- Gender field accepts "Male", "Female", or "Other"
- All wrestler fields except `name` and `gender` are optional
- The `is_user_created` field distinguishes between test data and user-created wrestlers
- Wrestlers are global resources available to all shows via show roster assignments
